Overview

The Code Difficulty Analyzer is designed to assist educators, problem setters, and developers in estimating the difficulty of a programming problem. Leveraging the power of LLMs, this tool evaluates problem attributes such as bitwise logic, constraints, sample test cases, and implementation complexity. This ensures consistency and transparency in tagging problems as Easy, Medium, or Hard.

This tool is particularly useful for:

  • Curating coding problem sets for platforms and contests
  • Providing a baseline difficulty estimate for new problems
  • Assisting educators in classroom or homework design

×

How to Use?

  1. Enter the Problem Title: Clearly summarize what the problem is about in 5-10 words (e.g., Bitmask Register Update).
  2. Describe the Problem: Include the question statement, functional expectations, any bit field or logical conditions, constraints, and at least one sample input/output pair.
  3. Submit for Analysis: Press "Analyze Difficulty". This sends the data to our LLM backend which predicts the difficulty and provides reasoning.
  4. Review Results: On the right panel, the difficulty (Easy/Medium/Hard) will be color-coded with a 2-3 sentence justification.
  5. Iterate: You can adjust the question or constraints and re-submit for a different analysis.
